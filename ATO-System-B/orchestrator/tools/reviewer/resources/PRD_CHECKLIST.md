## 4. Cross-Reference 검증

### 4.1 일관성 체크

| 체크 항목 | 검증 내용 |
|----------|----------|
| 용어 일관성 | PRD, IA, WF, SDD에서 동일 용어 사용 |
| ID 참조 | 기능 ID가 모든 문서에서 일관됨 |
| 데이터 필드 | IA의 데이터가 WF에서 바인딩됨 |
| API 연결 | WF의 액션이 SDD의 API와 연결됨 |

### 4.2 누락 체크

| 체크 항목 | 누락 시 조치 |
|----------|-------------|
| 기능 누락 | FAIL - 누락 기능 ID 보고 |
| 화면 누락 | FAIL - 필요 화면 명시 |
| API 누락 | WARN - API 추가 권고 |
| 데이터 누락 | WARN - 데이터 정의 권고 |

## 5. 검증 스코어 계산

### 5.1 점수 산정 공식

```
총점 = Σ(항목별 점수 × 가중치)

항목별 점수:
- PASS: 100점
- PARTIAL: 50점
- FAIL: 0점
```

### 5.2 합격 기준

| 구분 | 기준 |
|------|------|
| 전체 점수 | 80점 이상 |
| 필수 항목 | 100% PASS |
| HIGH 이슈 | 0개 |
