## 3. 데이터 흐름 (Data Flow)

```
[BOARD_MUZZIMA] ──┬──> [API: /api/podcast/best-posts] ──> [베스트 게시물 목록]
                  │
                  └──> [API: /api/podcast/script] ──> [팟캐스트 대본]
                                                           │
                                                           v
                                                    [PodcastPlayer UI]
                                                           │
                                                           v
                                              [사용자 재생 컨트롤] ──> [TTS 시뮬레이션]
```

### 3.2 실시간 데이터 소스

| 데이터 | 테이블/API | 업데이트 주기 |
|--------|----------|---------------|
| 베스트 게시물 | `BOARD_MUZZIMA` | 24시간 |
| 댓글 수 | `COMMENT` | 24시간 |
| 팟캐스트 대본 | `/api/podcast/script` | 요청 시 생성 |

## 4. 컨텐츠 구조

### 4.1 주요 정보 구성

```yaml
팟캐스트 대본:
  - speaker: HOST | GUEST
  - text: 대화 내용

메타데이터:
  - generatedAt: 생성 시간
  - wordCount: 단어 수
  - estimatedDuration: 예상 재생 시간

베스트 게시물:
  - BOARD_IDX: 게시글 ID
  - TITLE: 제목
  - READ_CNT: 조회수
  - AGREE_CNT: 추천수
  - COMMENT_CNT: 댓글 수
```

### 4.2 정보 우선순위

| 우선순위 | 항목 | 가중치 |
|----------|------|--------|
| 1 | 대본 (ScriptDisplay) | 50% |
| 2 | 재생 컨트롤 (PlayButton) | 30% |
| 3 | 메타 정보 | 15% |
| 4 | 베스트 게시물 목록 | 5% |

## 5. 상태 관리 (State Management)

### 5.1 상태 정의

```yaml
플레이어 상태:
  - idle: 초기 상태 (재생 전)
  - playing: 재생 중
  - paused: 일시정지
  - completed: 재생 완료

데이터 상태:
  - loading: 데이터 로딩 중
  - success: 데이터 로드 성공
  - error: 데이터 로드 실패
```

### 5.2 상태 전이 조건

| 현재 상태 | 이벤트 | 다음 상태 |
|----------|--------|----------|
| idle | 재생 버튼 클릭 | playing |
| playing | 일시정지 버튼 클릭 | paused |
| paused | 재생 버튼 클릭 | playing |
| playing | 대본 끝 도달 | completed |
| completed | 재생 버튼 클릭 | playing (처음부터) |

## 7. 모바일/반응형 고려사항

### 7.1 모바일 우선 설계
- 단일 컬럼 레이아웃
- 터치 친화적인 재생 버튼 (최소 44x44px)
- 대본 텍스트 가독성 확보 (최소 16px)

### 7.2 데스크톱 확장
- 2단 레이아웃 (플레이어 + 베스트 게시물 사이드바)
- 더 넓은 대본 영역
- 키보드 단축키 지원 (Space: 재생/일시정지)
