## 4. 컨텐츠 구조

### 4.1 팟캐스트 대본 구조

```yaml
대본 구조:
  - opening: 오프닝 멘트
  - topics:
    - topic_1:
        title: 게시물 제목 (마스킹 처리)
        host_comment: Host 코멘트
        guest_comment: Guest 코멘트
    - topic_2: ...
    - topic_3: ...
    - topic_4: ...
    - topic_5: ...
  - closing: 클로징 멘트
```

### 4.2 정보 우선순위

| 우선순위 | 항목 | 가중치 |
|----------|------|--------|
| 1 | 대본 (Host/Guest 대화) | 50% |
| 2 | 재생 컨트롤 | 25% |
| 3 | 베스트 게시물 목록 | 15% |
| 4 | 헤더/메타 정보 | 10% |

## 5. 상태 관리 (State Management)

### 5.1 상태 정의

```yaml
상태 목록:
  - LOADING: 데이터 로딩 중
  - READY: 재생 준비 완료
  - PLAYING: 재생 중
  - PAUSED: 일시 정지
  - ENDED: 재생 완료
  - ERROR: 에러 발생
```

### 5.2 상태 전이 조건

| 현재 상태 | 이벤트 | 다음 상태 |
|----------|--------|----------|
| LOADING | 데이터 로드 완료 | READY |
| LOADING | 에러 발생 | ERROR |
| READY | 재생 버튼 클릭 | PLAYING |
| PLAYING | 일시정지 클릭 | PAUSED |
| PAUSED | 재생 버튼 클릭 | PLAYING |
| PLAYING | 대본 끝 도달 | ENDED |
| ENDED | 재생 버튼 클릭 | PLAYING (처음부터) |

## 7. 모바일/반응형 고려사항

### 7.1 모바일 우선 설계

- 세로 스크롤 기반 레이아웃
- 터치 친화적 재생 버튼 (최소 48px)
- 대본 텍스트 가독성 확보 (16px 이상)

### 7.2 데스크톱 확장

- 2단 레이아웃 (좌: 플레이어, 우: 베스트 목록)
- 더 넓은 대본 표시 영역
- 키보드 단축키 지원 (Space: 재생/일시정지)
