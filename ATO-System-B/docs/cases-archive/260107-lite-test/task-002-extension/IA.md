## 4. 컨텐츠 구조

### 4.1 팟캐스트 대본 구성

```yaml
대본 구조:
  - 섹션: Intro / Main / Outro
  - 발화자: Host / Guest
  - 형식: 대화체 (구어체)
  - 분량: 450~500 단어 (3분)
```

### 4.2 발화자 역할 분담

| 발화자 | 역할 | 말투 |
|--------|------|------|
| **Host** | 진행, 질문, 공감 유도 | 친근하고 밝은 톤 |
| **Guest** | 분석, 의견 제시, 정보 전달 | 전문적이면서 편안한 톤 |

### 4.3 섹션별 시간 배분

| 섹션 | 시간 | 단어 수 | 목적 |
|------|------|---------|------|
| Intro | 30초 | ~75단어 | 오프닝, 토픽 예고 |
| Main Topic 1 | 45초 | ~110단어 | 핵심 토픽 (번아웃/이민) |
| Main Topic 2 | 45초 | ~110단어 | 서브 토픽 (직원 관리) |
| Main Topic 3 | 30초 | ~75단어 | 가벼운 전환 (유명인) |
| Outro | 30초 | ~75단어 | 마무리, 예고 |

### 4.4 정보 우선순위

| 우선순위 | 항목 | 가중치 |
|----------|------|--------|
| 1 | 공감 유도 토픽 (번아웃, 스트레스) | 40% |
| 2 | 실용 정보 (직원 관리 노하우) | 30% |
| 3 | 가벼운 토크 (유명인, 트렌드) | 20% |
| 4 | 메타 정보 (회차, 예고) | 10% |

## 5. 상태 관리 (State Management)

### 5.1 대본 생성 상태

```yaml
상태 목록:
  - IDLE: 대기 상태
  - ANALYZING: 분석 결과 로딩 중
  - GENERATING: 대본 생성 중
  - COMPLETED: 대본 생성 완료
  - ERROR: 오류 발생
```

### 5.2 상태 전이 조건

| 현재 상태 | 이벤트 | 다음 상태 |
|----------|--------|----------|
| IDLE | generate_click | ANALYZING |
| ANALYZING | data_loaded | GENERATING |
| GENERATING | script_created | COMPLETED |
| GENERATING | error_occurred | ERROR |
| ERROR | retry_click | ANALYZING |

## 6. PII 마스킹 규칙

### 6.1 마스킹 대상

| 유형 | 패턴 | 처리 방법 |
|------|------|-----------|
| 환자 정보 | 이름, 병명 언급 | 일반화 ("환자분") |
| 의사 식별 | 실명, 병원명 | 일반화 ("한 선생님") |
| 비속어 | 욕설, 비하 표현 | 순화 또는 생략 |

### 6.2 마스킹 예시

```
원본: "무찌마에서 A 선생님이 올린 글이 화제입니다"
변환: "무찌마에서 한 선생님이 올린 글이 화제입니다"

원본: "시X 직원들이..."
변환: "직원들이 답답할 때가..."
```
