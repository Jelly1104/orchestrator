# IA.md - 정보 구조

## 1. 사이트맵

```
오케스트레이터 검증 대시보드
├── 메인 대시보드 (/)
│   ├── 검증 상태 개요
│   ├── 파이프라인 진행도
│   └── 실시간 모니터링
├── Phase A: 분석 검증 (/analysis)
│   ├── SQL 쿼리 검증
│   ├── 스키마 일치성 검사
│   └── 결과 데이터 품질
├── Phase B: 설계 검증 (/design)
│   ├── IA 문서 검증
│   ├── Wireframe 검증
│   ├── SDD 검증
│   └── HANDOFF 검증
├── Phase C: 구현 검증 (/implementation)
│   ├── 코드 생성 검증
│   ├── 테스트 커버리지
│   └── 빌드 성공률
└── 시스템 메트릭 (/metrics)
    ├── HITL 체크포인트 로그
    ├── 재시도 현황
    └── 보안 감사 로그
```

## 2. 네비게이션 구조

### 2.1 메인 네비게이션
```
[대시보드] [분석 검증] [설계 검증] [구현 검증] [시스템 메트릭]
```

### 2.2 브레드크럼 패턴
```
홈 > Phase A > SQL 쿼리 검증 > 세그먼트 조회
홈 > Phase B > SDD 검증 > 레거시 매핑
홈 > Phase C > 코드 검증 > TypeScript 컴파일
```

## 3. 정보 계층 구조

### Level 1: 파이프라인 단계별 그룹핑
- Phase A (정량적 분석)
- Phase B (정성적 설계) 
- Phase C (코드 구현)

### Level 2: 검증 영역별 세분화
- 기능별 검증 (SQL, 문서, 코드)
- 품질 지표 (성공률, 커버리지, 일치율)
- 시스템 메트릭 (성능, 보안, HITL)

### Level 3: 상세 검증 항목
- 개별 산출물 상태
- 오류 상세 로그
- 수정 이력 추적

## 4. 데이터 흐름 매핑

```
PRD 입력 → Gap Check → 유형 판별 → 파이프라인 라우팅
    ↓
Phase A: SQL 생성 → 스키마 검증 → 쿼리 실행 → 결과 분석
    ↓
Phase B: IA 설계 → Wireframe → SDD → HANDOFF 생성
    ↓
Phase C: 코드 생성 → 테스트 작성 → 빌드 검증 → Output 검증
    ↓
최종 산출물 저장 및 품질 리포트 생성
```
