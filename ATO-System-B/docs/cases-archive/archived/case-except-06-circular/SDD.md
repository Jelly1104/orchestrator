# SDD.md - 시스템 설계

## 1. 아키텍처 개요
- **순환 구조 설계**: A, B, C 각 기능은 독립적 모듈로 구현
- **상태 관리 시스템**: 각 기능의 수행 상태를 별도로 추적

## 2. 데이터 흐름
- A → C: A의 처리 결과를 C로 전달
- B → A: B의 처리 결과를 A로 전달
- C → B: C의 처리 결과를 B로 전달

## 3. 의존성 관리
- 각 기능은 선행 기능이 완료된 후에만 작동
- 상태 점검 API를 통해 비동기 식으로 각 기능의 완료 여부 확인

## 4. API 설계
- **GET /api/v1/feature/a**: B의 결과를 입력으로 받아 C에 전달
- **GET /api/v1/feature/b**: C의 결과를 입력으로 받아 A에 전달
- **GET /api/v1/feature/c**: A의 결과를 입력으로 받아 B에 전달
