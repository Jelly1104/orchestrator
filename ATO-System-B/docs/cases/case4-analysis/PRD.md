# PRD: 활성 회원 패턴 분석 PoC (Case #4)

> **문서 버전**: 1.0.0
> **작성일**: 2025-12-17
> **상태**: Approved
> **참조 문서**: `DOCUMENT_PIPELINE.md`, `AI_Playbook.md`, `DOMAIN_SCHEMA.md`

---

## 1. 개요 (Overview)

- **목적**: HEAVY 세그먼트 회원(16,037명)의 프로필 패턴을 분석하여 AI Agent Use Case 우선 적용 대상 세그먼트를 정의
- **타겟 유저**: AI PM, 기획팀, CRM팀
- **핵심 지표**: 전체 대비 +5%p 이상 차이 세그먼트 발견

---

## 2. 배경

### 2.1 현황

| 항목 | 현황 | 문제점 |
|------|------|--------|
| Agent 사용률 | 5% 미만 | 홈페이지 강제 팝업에도 대부분 닫기 클릭 |
| Use Case | 불명확 | "왜 써야 하는지" 사용자가 모름 |
| 타겟 세그먼트 | 미정의 | 어떤 회원에게 먼저 적용할지 근거 없음 |

### 2.2 접근 방식: 성공 복제 (Offensive)

**핵심 질문**: "플랫폼을 잘 쓰는 사람들은 누구이고, 어떤 특징을 갖고 있는가?"

```
활성 회원 패턴 발견
       ↓
"개원의 + 내과 + 서울/경기 = 가장 활발" (예시)
       ↓
이 세그먼트 대상으로 Agent Use Case Trigger 우선 설계
       ↓
G1 목표 달성 가속 (5% → 15%)
```

## 3. 범위 (Scope)

**In Scope**:
- HEAVY 세그먼트 회원 (16,037명) 프로필 분석
- 행동 데이터(KPI 리포트)와 프로필 데이터(회원 DB) 조인
- 활성 회원 공통 패턴 도출 (전문과목, 근무형태, 지역)
- G1 Use Case Trigger 후보 세그먼트 제안

**Out of Scope**:
- 실시간 세그먼트 분류 API 개발
- 대시보드 UI 구현
- 휴면 회원 예측 모델 (Case #5에서 진행)

## 5. 분석 설계

### 5.1 2-Layer 분석 구조

```
┌─────────────────────────────────────────────────────────────┐
│  Layer 1: 행동 세그먼트 (WHY - 트리거 설계용)                 │
├─────────────────────────────────────────────────────────────┤
│  • 주 이용 서비스: COMMUNITY / CAREER / BOTH                │
│  • 이용 패턴: EXPLORER(다양) / FOCUSED(단일)                │
│  • 시간대: MORNING / AFTERNOON / EVENING / NIGHT            │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  Layer 2: 프로필 세그먼트 (WHO - 타겟팅용)                    │
├─────────────────────────────────────────────────────────────┤
│  • 근무형태: 개원 / 봉직 / 전공의                            │
│  • 전문과목: CODE_MASTER 참조                               │
│  • 가입 코호트: 신규 / 1년차 / 장기                          │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 분석 흐름

```
Step 1. 활성 회원 정의
  → KPI 리포트의 HEAVY 세그먼트 (16,037명) 추출
  → 조건: 최근 30일 내 6+ 세션/일

Step 2. 프로필 조인
  → HEAVY 회원 U_ID → USERS → USER_DETAIL 조인
  → 필터: U_KIND = 'DOC001' (의사만)

Step 3. 패턴 분석
  → 전문과목별 분포 vs 전체 회원 분포 비교
  → 근무형태별 분포 vs 전체 회원 분포 비교
  → 가입 코호트별 활성률 비교

Step 4. 인사이트 도출
  → "어떤 세그먼트가 과대 대표(Over-represented) 되는가?"
  → G1 Use Case Trigger 우선 적용 대상
```
