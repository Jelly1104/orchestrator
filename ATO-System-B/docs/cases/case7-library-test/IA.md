# IA.md - 활성 사용자 현황 대시보드 정보 구조

## 1. 라우팅 구조

```
/admin
└── /users
    └── /dashboard  ← 활성 사용자 현황 대시보드
        ├── /overview     (전체 현황)
        ├── /statistics   (통계 분석)
        └── /trends       (추이 분석)
```

## 2. 페이지 계층 정의

### 2.1 메인 대시보드 (/admin/users/dashboard)

```
Header
├── 페이지 제목: "활성 사용자 현황 대시보드"
├── 마지막 업데이트 시간
└── 새로고침 버튼

Main Content
├── KPI Cards Section
│   ├── 전체 사용자 수
│   ├── 활성 사용자 수 [⚠️ AI 추론됨 - 검토 필요]
│   ├── 신규 가입자 (당월)
│   └── 증감율 (전월 대비)
│
├── Charts Section
│   ├── 월별 신규 가입자 추이 (Line Chart)
│   └── 유형별 사용자 분포 (Pie Chart)
│
└── Data Table Section
    └── 상세 사용자 현황 테이블
```

## 3. 정보 우선순위

| 우선순위 | 정보 | 비즈니스 중요도 |
|----------|------|----------------|
| 1 | 전체 사용자 수 | Critical - 전체 규모 파악 |
| 2 | 월별 신규 가입 추이 | High - 성장성 지표 |
| 3 | 유형별 분포 | High - 사용자 구성 분석 |
| 4 | 활성 사용자 수 | Medium - 정의 기준 불명확 |

## 4. 탐색 흐름

```
대시보드 진입 → KPI 확인 → 차트 분석 → 상세 데이터 확인
     ↓
상황별 액션:
- 신규 가입 급증/급감 → 마케팅팀 알림
- 특정 유형 급증 → 서비스 운영팀 검토
```

## 5. 권한 및 접근 제어

- **관리자**: 전체 데이터 조회 가능
- **운영팀**: 통계 데이터만 조회 가능
- **마케팅팀**: 신규 가입 추이만 조회 가능
