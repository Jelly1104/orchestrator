## 🏢 1. 팀 정의 (Team Definition)

### 미래전략실 (AI Transformation Office)

메디씨앤씨의 핵심 서비스(의사 커뮤니티, 경력관리 H-Link)의 **레거시 구조·업무 방식·개발 프로세스를 AI 중심으로 전면 재설계**하는 전사 혁신 조직입니다.

### 두 가지 축

| 역할 | 핵심 미션 | 키워드 |
|------|----------|--------|
| **AX PM** (AI Experience) | 사용자 문제 정의 → AI 기술로 실제 경험 설계 | UX/UXR, 도메인 지식, Consumer-facing |
| **AB PM** (AI Builder) | AI 모델·백엔드·인프라, ML 엔지니어링 | 모델 훈련, 품질 관리, 데이터 설계 |

---

## 🌟 2. 핵심 목적

### 1) 레거시 개발 구조·업무 프로세스 혁신

```
기존: 기획 → 디자인 → 개발 → QA (순차적, 병목 발생)
     ↓
혁신: 기획·디자인·개발·QA 동시 진행 (병렬 + 자동화)
```

### 2) 의사 커뮤니티/H-Link 도메인 특화 AI 구축

- 메디게이트 전문 지식 (의료 도메인)
- 의사 경력 데이터 (H-Link) 구조적 특징
- 의사 회원 UX 패턴

### 3) 회사 전체의 AI 생산성 향상

모든 팀이 AI 도구를 실무에 활용할 수 있도록 **템플릿·프로세스·모델·내부 도구**를 만들고 전파

### 4) 기술팀과 비기술팀 사이 연결 허브

비기술 조직(서비스/기획/영업/운영)과 기술 조직(BE/FE/AI/데이터)을 연결해 AI 기반의 신속한 문제 해결

## 🔄 4. Operating Model

### 3-Layer 구조

```
[AX PM]  사용자 문제 정의 → UX Flow → AI 기능 요구사항
        ↓
[AB PM]  모델 스코프 → 데이터 설계 → 품질 기준 정의 → API 스펙
        ↓
[FE/BE]  기능 개발 → 통합 → 배포 → 모니터링
```

### 병렬 협업 방식

| 기존 방식 (레거시) | 미래전략실 방식 (AI 기반) |
|------------------|------------------------|
| 기획 → 디자인 → 개발 → QA | 기획·UXR·프로토타입·디자인·개발·QA **동시 진행** |
| 각 단계가 다음 단계의 시작 조건 | AI 기반 **단일 문서·단일 스펙** 위에서 동시 실행 |

## 📜 6. 팀 운영 원칙

1. **AI First** - 모든 업무는 AI로 1차 생성 → 사람이 검토
2. **병렬 협업이 기본** - 기획-디자인-개발은 "동시 진행"
3. **단일 스펙 문서 체계** - 전사 모든 팀이 하나의 AI 생성 문서에서 협업
4. **빠른 실험·빠른 폐기** - 프로토타이핑 24시간 내 생성 → 48시간 내 의사결정
5. **데이터 기반 개선** - 감이 아니라 사용자·로그 기반 판단
6. **표준화된 템플릿 사용** - 문서·모델·API·디자인은 자동 생성 템플릿 기반
7. **재사용(Reuse) 우선** - 이미 만든 AI Layer는 모든 팀이 공유·활용
8. **레거시보다 효율 우선** - "기존 방식"보다 "더 빠르고 정확한 방식"
