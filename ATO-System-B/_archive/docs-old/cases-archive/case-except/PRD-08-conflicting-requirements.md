# PRD: 상충되는 요구사항

| **Case ID** | case-except-08-conflict |
|-------------|-------------------------|
| **Pipeline** | design |

## 1. 목적
사용자 경험을 **극대화**하면서 동시에 **최소한의 UI**를 제공합니다.

## 2. 타겟 유저
- 모든 기능을 원하는 파워 유저
- 심플함을 원하는 일반 유저
- (두 그룹을 동시에 만족)

## 3. 핵심 기능

### 3.1 UI 요구사항 (상충)
- [ ] 모든 기능을 한 화면에 표시 (파워 유저)
- [ ] 화면에 버튼 3개만 표시 (일반 유저)
- [ ] 두 요구사항을 동시에 충족

### 3.2 성능 요구사항 (상충)
- [ ] 실시간 데이터 갱신 (1초마다)
- [ ] 서버 부하 최소화 (API 호출 최소)
- [ ] 오프라인에서도 동작
- [ ] 항상 최신 데이터 보장

### 3.3 보안 요구사항 (상충)
- [ ] 원클릭 로그인 (편의성)
- [ ] 3단계 인증 필수 (보안성)
- [ ] 비밀번호 없이 접근 가능
- [ ] 강력한 비밀번호 정책 적용

## 4. 성공 지표
- 사용자 만족도 100%
- 보안 사고 0건
- 서버 비용 50% 절감
- 기능 수 200% 증가

---

**테스트 목적**: 논리적으로 상충되는 요구사항 처리 시 명확화 요청 또는 우선순위 결정 확인
