### Agenda : 회원 리뉴얼 1차·2차 범위 정리 및 미래전략실 협업 포인트 논의

---

## **1. 배경**

- 2025년 회원 시스템 리뉴얼 프로젝트는 **기술 인프라 분리(프론트/백)** 목적으로 시작했으나, 실제 운영·마케팅 니즈가 서비스 전반에 존재함이 확인됨.
- 플랫폼 매출이 큰 서비스의 개선이 완료된 이후, **사용자 타겟팅·마케팅 효과성**을 높이기 위해 회원 데이터 구조를 우선 개편해야 하는 필요가 대두됨.
- 미래전략실 역시 **유저 프로파일·타겟팅 지표**에 대한 요구가 있어 초기 단계부터 방향성 맞추기 위해 본 미팅을 진행.

---

## **2. 목적**

1. [회원 리뉴얼 **1차 범위(인프라·데이터 표준화)** 확정](https://www.figma.com/design/4YyBvPrFX2XW6ul5GIEwdV/A-1-1-MEDI-GATE---%ED%9A%8C%EC%9B%90%EA%B0%80%EC%9E%85-%EB%A1%9C%EA%B7%B8%EC%9D%B8?node-id=4012-17688&t=skwg4wmoH4nTjVFl-1)
2. 2차 이후 협업해야 할 **데이터/타겟팅/캠페인 구조 설계** 논의
3. 기존 회원 유형·입력 필드·기관 분류의 문제점 정리 및 개선 방향 논의
4. 미래전략실이 필요한 **정확한 데이터 수집·지표 설계 포인트 식별**

    - [분산된 회원 정보(근무처, 전문과, 기관코드 등)의 연계 및 데이터 최신화 체계 마련](https://docs.google.com/spreadsheets/d/17J2z8ITFL8811uSTTyxf_BBNuCpGE0sg_azV8M4nyYg/edit?usp=sharing)

    - 행동 기반 지표 수집 체계 구축

    - 타겟팅 캠페인 및 마케팅 자동화 연동
- **3차 목표:**
    - 유저 단위 마케팅 채널 효과 분석

## **4. 상세**

### **4-1. 리뉴얼 전 사용자/데이터 구조 문제점 정리**

### **(1) 접근성·사용성 문제**

- 비의사 회원의 **모바일 접근 불가**, 모바일 → PC 화면으로 강제 전환
    - e.g) 병의원, 기업, 의대생, 학회, 일반회원
- 가입 단계에서 **과도한 항목 수집**(예: 기업회원에게 자택전화번호 요구)
- 회원 유형별 UI/UX·권한 구분 불명확 => 혼선 발생

### **(2) 회원 정보가 서비스별로 분산 관리**

- 동일 정보가 각 서비스(초빙, H링크, 오토원 등)마다 다르게 저장
- 최신 정보가 특정 서비스에만 존재하여 **회원 프로파일이 업데이트되지 않음**
- 결제·핵심 정보가 서비스 단위에만 머물러 전역 회원 시스템에 반영되지 않음

### **(3) 기관/병원 분류 기준의 불일치**

- 회원가입: 심평원 기준
- 초빙정보: 초빙 자체 분류
- H링크: 또 다른 분류 사용
    
    → **실제 시장 대비 기관 수가 부풀려지는 오류 발생**
    
    → 기관 단위 데이터 기반 분석/타겟팅이 불가능
    

### **(4) 마케팅 타겟팅 및 지표 수집의 한계**

- 나이/근무처 등 기본 정보 외에 **행동 기반 지표 수집 불가**
- 채널별 성공/반응률은 있으나 **유저 단위 매칭 불가 → 캠페인 효과 분석 불가능**
- 전문과·세부전공·진료과목 등 분류값 부정확, 업데이트 체계 부재

---

### **4-2. 1차 리뉴얼 범위(2024 Q4~2025 Q1 예정)**

- **회원가입 6개 유형의 가입 절차 정비**
- 로그인/아이디찾기/비밀번호찾기 프로세스 표준화
- 로그인 직후 노출되는 체험 정보 개선 → 자동 업데이트 흐름 유도
- 회원정보 페이지를 레이어 팝업 → **‘마이페이지’ 독립 메뉴**로 전환
- 분산된 회원 정보 연동 / 최신 정보 자동 업데이트
- 심평원 기관 코드 기준 매핑 기본 틀 구축
- 모바일/PC 백엔드 구조 통합 기획 (테스트 중복 문제 해결 목적)

---

### **4-3. 2차 리뉴얼 논의 사항 (미래전략실 협업 핵심)**

### **(1) 신규 필드 추가 여부**

- 전문과·세부전공·진료과목 구조 재설계
- 사용자 입력 기반 선호 정보(서비스 관심사 등) 추가 가능
- 다만 신규 필드 추가 시, 기존 **13만 가입자 데이터 업데이트 캠페인 필수**

### **(2) 전문과/세부전공 구조 문제**

- 현재는 내과만 세부과 존재
- 실제 의료계는 **내과·외과·소청과 등 최소 3계열로 존재**
- 의료현장에서 사용하는 세부전공 기준이 필요
    
    → 상급종합병원 진료과목 태그 수집 후 LLM 기반 정규화 제안됨
    
    → 미래전략실이 1차 매핑 생성 → 기획팀 리뷰
    

### **(3) 근무처·병원명 코드 문제**

- 기존에는 **표시명(text)**만 저장
- 병원명이 바뀌어도 사용자 프로필은 갱신되지 않음
- 병원 폐업/재등록 여부 관리 불가
    
    **→ 요양기관번호(YKIHO) 기반 통일 관리 필요**
    

### **(4) 주소(거주지) 정보 논쟁**

- 기획팀: 개인정보 최소화, 회원정보에서 상세주소 제외 희망
- 초빙팀/미래전략실: 거리 기반 매칭 정확도를 위해 **동·세부주소 필수**
- 결론: 일단 기존 형태 유지, 추후 **이력서 주소와 동기화 정책 검토**

### **(5) 회원 유형/서비스 접근권한 재정의 필요**

- 일반회원 → 임대분양 접근 불가 등 비합리적인 구조 존재
- 회원 유형 변경 기능 필요성 제기
- 신규 사용자가 ‘어떤 서비스 이용하려는지’ 먼저 선택하면
    
    → 자동으로 적합한 유형 안내하는 UX 제안됨
    

---

### **4-4. 3차 논의 예상 범위**

- 캠페인 기반 타겟팅 시스템 설계
- 마케팅 채널·사용자 이벤트 기반의 지표 정교화
- 유저 단위 효과 측정 모델 구축
- 통합 로그인/생체인증 적용
- 회원 가입/로그인에 **카카오·네이버·구글 SSO** 도입 요청

요약

### 배경 및 목적

- 기존의 단순 기술적 리뉴얼에서 확장하여 마케팅 타깃팅과 유저 프로파일 관련 고민 반영
- 2월부터 시작된 프로젝트로 초기에는 기술적 분리 목적이었으나 운영 부서의 니즈가 반영되어 확장됨
- 플랫폼 매출 서비스(임대, 분양) 리뉴얼 후 마케팅/사업 부서 주관 서비스를 위한 회원 리뉴얼 진행

### 현재 회원 관련 주요 이슈

- **접근성/사용성 문제**: 의사 외 회원 유형의 모바일 접근 불가, 모바일 가입 프로세스 미비
- **정보 관리 분산**: 서비스별로 정보가 분산되어 최신 업데이트 정보가 전체 시스템에 반영되지 않음
- **데이터 분류 기준 불일치**: 기관 분류가 서비스마다 다름 (심평원 분류, 초빙 분류, H-링크 분류 등)
- **마케팅 타깃팅 한계**: 행동 기반 지표를 활용한 타깃팅 확장이 불가능, 유저 단위 마케팅 채널 효과 분석 제한

### 리뉴얼 단계별 계획

- **1단계 (4분기)**: 회원 가입 절차 정리, 분산된 정보 연계, 분류 기준 표준화, API 인증 통일
- **2단계**: 모바일 관련 백엔드 통합, 결제 정보 및 행동 분석 지표 통합, 회원 타깃팅 필요 분류값 정리 및 업데이트
- **3단계**: 실질적인 유저 단위 캠페인 진행

### 1차 리뉴얼 범위

- **진료 과목**: 새로 추가된 항목으로, 전문의 외에도 실제 진료하는 과목 선택 가능
- 회원 가입 프로세스 (6개 가입 유형)
- 로그인 및 아이디/비밀번호 찾기 프로세스
- 체험 정보 관련 업데이트
- 회원 정보를 마이페이지 형태로 통합

### 회원 정보 항목 변경사항

- **기본 정보**: 큰 변경사항 없음
- **관심 서비스**: 개원 입지, 연수 강사 등 새로 추가된 서비스 항목 추가
- **전문 과목**: API를 통해 자동으로 불러오기 기능 구현
- **세부 전문 과목**: 내과 외에도 소아청소년과, 외과 추가
- **거주 지역**: 현재 상세 주소까지 수집하는 형태 유지 (초빙 서비스에서 위치 기반 매칭에 활용)

### 병원/기관 정보 관련 이슈

- 현재 병원 정보가 코드가 아닌 이름 기반으로 관리되는 문제
- 요양기관 번호 기반으로 관리하는 방안 논의
- 심평원 코드 변경 시 기존 사용자 정보 업데이트 방안 필요
- 병원 분류는 심평원 기준으로 통일하되, 서비스별로 필요한 추가 분류는 2단계로 관리

### 기타 논의 사항

- 임상/기초의학 구분 문제: 현재 구조가 모든 의사가 임상으로 가정하고 기초의학을 추가 선택하는 방식으로 설계되어 있어 개선 필요
- 회원 정보와 서비스별 정보(이력서 등) 간의 연동 방안
- 개인정보 최소화와 서비스 필요성 간의 균형
