## 1. 목적 (Objective)

### 1.1 비즈니스 목표
메디게이트 의료진 회원들에게 **개인화된 채용공고를 자동 추천**하여 B2C 채용 매출을 증대시킵니다.

### 1.2 문제 정의
- 현재: 의료진이 수동으로 채용공고를 검색해야 함 (낮은 engagement)
- 목표: AI 기반 개인화 추천으로 매칭률과 지원 전환율 향상

## 2. 타겟 유저 (Target User)

### Primary Persona: 경력 의사 (Dr. 김철수, 35세)
- **현재 상황**: 개원 준비 중, 수련병원에서 봉직 경험 5년
- **Pain Point**: "내 전공과 경력에 맞는 좋은 자리를 놓치고 싶지 않은데, 매일 확인하기 번거로워"
- **목표**: 내과 전문의 채용공고 중 연봉 1억 이상, 서울/경기 지역 우선

### Secondary Persona: 신규 간호사 (김영희, 26세)  
- **현재 상황**: 간호학과 졸업 후 첫 취업 준비
- **Pain Point**: "큰 병원부터 의원까지 어디가 나에게 맞을지 모르겠어"
- **목표**: 대학병원 또는 종합병원, 교육 프로그램이 잘 되어 있는 곳

## 3. 핵심 기능 (Core Features)

### F1. 개인화 추천 피드
- **입력**: 회원 프로필 (전공, 경력, 희망 지역, 연봉)
- **출력**: 매칭 점수 기반 채용공고 리스트 (상위 10개)
- **갱신**: 일 1회 자동 업데이트

### F2. 매칭 점수 알고리즘
```
매칭 점수 = (전공 매칭 40%) + (경력 매칭 25%) + (지역 매칭 20%) + (연봉 매칭 15%)
- 전공 매칭: 정확히 일치 100점, 유사 전공 70점, 전과목 50점
- 경력 매칭: 요구 경력 ±1년 100점, ±2년 80점, ±3년 60점
```

### F3. 실시간 알림
- 높은 매칭률 공고 등록 시 푸시 알림 (95점 이상)
- 마감 임박 알림 (D-3, 매칭 점수 80점 이상)

### F4. 관심 공고 관리
- 북마크 기능
- 지원 현황 추적
- 유사 공고 추가 추천

## 4. 성공 지표 (Success Criteria)

### Primary KPI
- **추천 클릭률(CTR)**: 15% 이상 (현재 검색 기반 5% 대비 3배 향상)
- **지원 전환율**: 8% 이상 (추천 피드에서 실제 지원까지)

### Secondary KPI  
- **일일 활성 사용자(DAU)**: 기존 대비 25% 증가
- **매출 기여도**: 추천 기반 지원이 전체 채용 매출의 30% 차지
- **사용자 만족도**: NPS 60점 이상

## 6. IA (Information Architecture)

### 6.1 사용자 플로우
```
1. 로그인 → 2. 프로필 설정 → 3. 추천 피드 → 4. 상세보기 → 5. 지원/북마크
                    ↓
              6. 알림 설정 → 7. 관심공고 관리
```

### 6.2 정보 구조
```
메인 네비게이션
├── 추천 피드 (기본 화면)
├── 관심 공고
├── 지원 현황
└── 설정
    ├── 알림 설정
    ├── 추천 기준 설정
    └── 프로필 관리
```

## 8. 기술 요구사항 (Technical Requirements)

### 8.1 성능
- API 응답 시간: 500ms 이내
- 추천 알고리즘 실행: 1초 이내
- 동시 사용자: 1,000명 처리 가능

### 8.2 보안
- JWT 기반 인증
- 개인정보(연봉, 지역 등) 암호화 저장
- HTTPS 필수

### 8.3 호환성
- 모바일 우선 (iOS/Android 웹뷰)
- 데스크톱 반응형 지원
